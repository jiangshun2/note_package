一、今日学习内容汇总

1. static 关键字的修饰范围

知识点：

  • 静态变量（static variables）：也叫类变量，属于类本身，而不是某个对象实例
  • 静态方法（static methods）：属于类，可以直接通过类名调用
  • 静态代码块（static blocks）：用于类加载时执行一次的初始化操作
  • 静态内部类（static nested classes）：嵌套在另一个类中的静态类
  • 静态导入（static import）：如 import static java.lang.Math.*;

实际应用场景：

  • ✅ 静态变量：用于共享状态（如统计对象创建次数）
  • ✅ 静态方法：工具方法（如 Math.random()）
  • ✅ 静态代码块：初始化静态资源、加载驱动
  • ✅ 静态内部类：逻辑上属于外部类但不依赖实例

2. static 方法不能访问非静态成员的原因

知识点：

  • 非 static 的成员变量和方法属于某个具体的对象实例，只有在创建对象（new）之后才存在
  • static 方法或变量属于类本身，在类被加载到 JVM 时就存在了，不需要创建对象
  • 当 static 方法被调用时，可能还没有任何该类的对象被创建，因此无法访问非静态成员

实际应用场景：

  • ✅ 避免在静态方法中直接访问实例变量
  • ✅ 如需访问实例成员，需要先创建对象实例

3. static 变量的应用：统计对象创建次数

知识点：

  • 使用 static 变量作为计数器，因为它是类级别的，所有对象共享

知识点：

  • 静态代码块在类被加载到 JVM 时执行，而不是在创建对象时
  • 只执行一次，即使没有创建任何对象
  • 早于 main 方法和构造方法执行

  • ✅ 初始化静态资源（如加载配置文件）
  • ✅ 加载数据库驱动
  • ✅ 设置全局配置

5. main 方法必须是 static 的原因

知识点：

  • JVM 在启动程序时，并没有该类的任何实例对象
  • 当运行 java MyClass 时，JVM 会加载 MyClass 类并直接调用 public static void main(String[] args) 方法
  • 如果 main 方法不是 static 的，JVM 无法知道该用哪个对象来调用它

实际应用场景：

  • ✅ 所有 Java 应用程序的入口点

6. 代码块的执行顺序

知识点：

  • 静态代码块：类加载时执行，只执行一次
  • 构造代码块（{}）：每次创建对象时执行，在构造方法之前
  • 构造方法：每次创建对象时执行，在构造块之后

二、今日记忆口诀汇总

  1. static 修饰范围：

    static 修饰五大件，变量方法代码块
    内部类和静态导，类级共享是关键

  2. static 方法访问限制：

    静态方法无实例，非静成员不能用
    类加载时就存在，对象实例还未生

  3. static 变量应用：

    静态变量共享用，对象计数最典型
    构造方法自增一，类名调用取统计

  4. static 代码块：

    静态代码块加载时，只执行一次记心间
    早于 main 和构造，初始化资源最适合

  5. main 方法必须 static：

    main 方法是入口，JVM 调用无对象
    static 关键字不可少，否则启动就报错

  6. 代码块执行顺序：

    静态块先执行，构造块在构造前
    静态只执行一次，每次 new 都执行构造块
