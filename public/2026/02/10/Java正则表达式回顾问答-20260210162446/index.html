一、今日学习内容汇总

1. 正则表达式的核心类

知识点：

  • Pattern 类：用于编译正则表达式，通过 Pattern.compile(String regex) 方法
  • Matcher 类：用于对目标字符串执行匹配操作，通过 pattern.matcher(CharSequence input) 获得
  • 所在包：java.util.regex

实际应用场景：

  • ✅ 编译正则表达式并复用 Pattern 对象
  • ✅ 对字符串执行匹配、提取、替换等操作

2. Pattern.compile() 的常见错误

知识点：

  • 正则表达式语法错误：如缺少右括号，会抛出 PatternSyntaxException
  • 忘记转义特殊字符：Java 字符串本身需要转义，正则表达式中某些字符也有特殊含义
  • 在循环中重复编译：性能问题，应该提前编译一次
  • 使用不支持的正则特性：如可变长度的后瞻断言
  • 忽略大小写等标志位：需要传入 Pattern.CASE_INSENSITIVE 等标志

实际应用场景：

  • ✅ 确保正则表达式语法正确
  • ✅ 避免在循环中重复编译正则
  • ✅ 正确设置标志位

3. 判断字符串是否包含数字

知识点：

  • 方法一：使用 Pattern + Matcher.find()
  • 方法二：使用 String.matches()（需注意全匹配）

实际应用场景：

  • ✅ 验证字符串是否包含数字
  • ✅ 表单验证

4. String.matches() 与 Pattern + Matcher 的区别

知识点：

  • 功能：两者返回值相同（都做全匹配）
  • 性能：String.matches() 每次都会编译正则，性能较差
  • 灵活性：Pattern + Matcher 可以复用 Pattern，且支持 find()、lookingAt() 等多种匹配方式

实际应用场景：

  • ✅ 简单、一次性判断：使用 String.matches()
  • ✅ 复杂、多次使用：使用 Pattern + Matcher

二、今日记忆口诀汇总

  1. 核心类：

    正则核心两兄弟，Pattern 和 Matcher
    util.regex 包中找，编译匹配全靠它

  2. Pattern.compile 错误：

    正则语法要正确，特殊字符要转义
    循环编译性能差，标志位设置别忘记

  3. 判断包含数字：

    判断包含用 find，全匹配用 matches
    正则 \d 来匹配，简单高效又直接

  4. 提取内容：

    提取内容用 group，捕获组加括号
    find 循环多次取，复杂信息轻松拿

  5. 替换操作：

    替换用 replaceAll，正则匹配要精准
    敏感信息可脱敏，日志清洗好帮手

  6. 邮箱提取：

    邮箱正则简化版，\w+@\w+.\w+
    实际场景要健壮，字符范围需扩展

  7. 日志提取：

    日志提取用多组，可选字段加问号
    业务逻辑来格式化，结构化信息轻松得
