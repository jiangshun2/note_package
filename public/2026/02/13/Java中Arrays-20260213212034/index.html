一、今日学习内容汇总

1. Arrays 类的包和作用

知识点：

  • 包路径：java.util.Arrays
  • 主要作用：提供了一系列静态方法，用于操作数组（如排序、查找、比较、填充、复制、转字符串等）
  • 特点：方法都是静态的，不能 new，也不能继承（它是 final 类）

实际应用场景：

  • ✅ 调试日志：用 Arrays.toString(arr) 快速打印数组内容
  • ✅ 数据预处理：用 Arrays.sort() 对用户输入的分数数组排序
  • ✅ 判等：用 Arrays.equals(arr1, arr2) 判断两个配置数组是否一致
  • ✅ 二分查找：在已排序的 ID 列表中用 Arrays.binarySearch() 快速定位用户

2. Arrays.asList() 的作用和陷阱

知识点：

  • 作用：将固定数量的元素（或一个数组）转换为一个 List 视图
  • 返回类型：java.util.Arrays.ArrayList（注意：不是 java.util.ArrayList）
  • 关键陷阱：
    • ❌ 不支持增删操作：调用 add() 或 remove() 会抛出 UnsupportedOperationException
    • ❌ 与原始数组双向绑定：修改原数组，List 会变；反之亦然
    • ❌ 基本类型数组会整体视为一个元素：int[] nums = {1, 2, 3}; List<int[]> list = Arrays.asList(nums); // list.size() == 1

正确使用方式：

  • 快速创建不可变的小型列表（如配置项、枚举值集合）
  • 配合 new ArrayList<>(Arrays.asList(...)) 创建可变副本

3. Arrays.equals() vs == vs Arrays.deepEquals()

知识点：

  • ==：比较两个数组引用是否指向同一块内存地址（即是否是同一个对象）
  • Arrays.equals()：逐个比较元素的值是否相等（对基本类型比值，对引用类型调用其 equals() 方法）
  • Arrays.deepEquals()：用于深度比较多维数组（递归比较内容）

实际应用场景：

  • ✅ Arrays.equals()：校验配置数组、缓存键生成、单元测试断言
  • ✅ Arrays.deepEquals()：比较矩阵数据（如图像像素、游戏地图）、序列化/反序列化后的数据一致性校验

4. 综合应用：数组操作实战

知识点：

  • 排序：Arrays.sort()
  • 查找：Arrays.binarySearch()（前提是有序）
  • 复制：Arrays.copyOf()、Arrays.copyOfRange()
  • 转换：Arrays.toString()
