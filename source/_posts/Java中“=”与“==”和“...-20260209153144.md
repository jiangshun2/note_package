## 一、今日学习内容汇总

### 1. equals() 方法

**知识点**：
- **默认实现**：`public boolean equals(Object obj) { return (this == obj); }`（比较引用是否相同）
- **重写目的**：根据对象的内容（而不是引用）来判断两个对象是否“相等”
- **必须遵守的约定**：
  - 自反性：`x.equals(x)` 必须为 true
  - 对称性：如果 `x.equals(y)` 为 true，那么 `y.equals(x)` 也必须为 true
  - 传递性：如果 `x.equals(y)` 为 true，且 `y.equals(z)` 为 true，那么 `x.equals(z)` 也必须为 true
  - 一致性：多次调用 `x.equals(y)` 应该始终返回相同结果
  - 非空性：对于任何非 null 的引用 x，`x.equals(null)` 必须返回 false

**实际应用场景**：
- ✅ 比较两个对象的内容是否相同（如用户登录时比较用户名密码）
- ✅ 在集合（如 HashMap、HashSet）中判断元素是否存在

### 2. hashCode() 方法

**知识点**：
- **作用**：返回对象的哈希码（整数），用于哈希集合（如 HashMap、HashSet）的快速定位
- **默认行为**：基于内存地址（JVM 实现相关）
- **重要约定**：如果两个对象 `equals()` 返回 true，它们的 `hashCode()` 必须相等

**实际应用场景**：
- ✅ 作为 HashMap 的 key 时，确保相同内容的对象能正确存储和查找
- ✅ 提高 HashSet 中元素查找的效率

### 3. toString() 方法

**知识点**：
- **默认实现**：`getClass().getName() + "@" + Integer.toHexString(hashCode())`（如 `Person@1b6d3586`）
- **重写目的**：返回对象的关键字段信息，便于调试、日志打印
- **正确实现**：返回包含关键字段的字符串，而不是打印

**实际应用场景**：
- ✅ 调试时快速查看对象的属性值
- ✅ 日志记录中输出对象的详细信息

### 4. clone() 方法

**知识点**：
- **作用**：创建并返回对象的副本
- **默认行为**：浅克隆（基本类型字段复制值，引用类型字段复制地址）
- **实现步骤**：
  1. 类必须实现 `Cloneable` 接口（标记接口，无方法）
  2. 重写 `clone()` 方法，改为 public
  3. 如果需要深克隆，在 `clone()` 中手动克隆引用类型的字段

**实际应用场景**：
- ✅ 创建对象的副本，避免修改原对象
- ✅ 深克隆复杂对象，确保引用类型字段也被独立复制

### 5. 其他重要方法

**知识点**：
- **getClass()**：返回运行时类对象（Class<?>），由 JVM 实现，不能重写
- **wait() / notify() / notifyAll()**：线程间协作方法，必须在 synchronized 块中调用
- **finalize()**：对象被 GC 前的清理钩子，JDK 9+ 已废弃，不推荐使用

**实际应用场景**：
- ✅ `getClass()`：用于反射、泛型类型检查
- ✅ `wait() / notify()`：线程间同步（现代 Java 推荐使用 `java.util.concurrent` 包中的工具）
